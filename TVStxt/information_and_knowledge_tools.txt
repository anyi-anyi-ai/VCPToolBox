VCP主动调用插件分类说明（信息与知识类）

一、说明目标
本文件用于提供从互联网、特定数据库中抓取数据的主动查找机制，当 AI 对本地记录毫无头绪，或需要引用实时的权威文献时，应当考虑使用此类工具爬取与检索。

二、统一调用规范
<<<[TOOL_REQUEST]>>>
maid:「始」署名「末」,
tool_name:「始」插件名「末」,
参数:「始」值「末」
<<<[END_TOOL_REQUEST]>>>

三、受支持的插件列表与具体格式

【1. TavilySearch】 (智能联网综合搜索)
- tool_name: TavilySearch
- 功能定位: API驱动的高级、防爬拦截综合搜索与聚合工具，可按主题与深度搜索当今时事与技术动态。
- 调用格式:
<<<[TOOL_REQUEST]>>>
tool_name:「始」TavilySearch「末」,
query:「始」检索词汇(最好附带今天的年份或具体时间要求)「末」,
search_depth:「始」advanced(推荐)「末」,
days:「始」最近几天的时间范围限制(如7)「末」,
max_results:「始」10「末」
<<<[END_TOOL_REQUEST]>>>

【2. UrlFetch】 (任意网页/本地文本与图片提取超级工具)
- tool_name: UrlFetch
- 功能定位: 通能的网址 URL 和本地 file:/// 路径内容读取工具，并可附带页面截取功能。
- 调用格式:
<<<[TOOL_REQUEST]>>>
tool_name:「始」UrlFetch「末」,
url:「始」网页链接或本地绝对文件路径「末」,
mode:「始」'text' (默认，提取纯文本) 或 'snapshot' (长截图并转码) 或 'image' (纯图片)「末」
<<<[END_TOOL_REQUEST]>>>

【3. BilibiliFetch】 (Bilibili综合信息获取器)
- tool_name: BilibiliFetch
- 功能定位: 获取B站视频详细信息，包含热评、所有字幕(支持CC机器与总结)、AI弹幕统计及视频截图。
- 参数： 
  - url: 视频 BV / 分 P 或者是普通页面链接长链均可。
  - need_subs: 默认true，尝试自动提纯字幕给AI做长视频总结。
  - action=search 时配合 keyword 可进行全B站全网视频搜。
- 调用格式:
<<<[TOOL_REQUEST]>>>
tool_name:「始」BilibiliFetch「末」,
url:「始」https://www.bilibili.com/video/BVxxx...「末」,
hd_snapshot:「始」true「末」
<<<[END_TOOL_REQUEST]>>>

【4. NCBIDatasets】 (生物学数据库NCBI快速访问)
- tool_name: NCBIDatasets
- 功能定位: NCBI Datasets v2 级联查询引擎，提供精确的学术物种、基因编号及组装级信息查询并拉回。
- 关键命令(command): 
  - search_genomes (依赖 tax_id 获取全物种对应基因组)
  - get_gene_info (按名称 gene_symbol 及物种名 organism 抽取基因表述)
- 调用格式:
<<<[TOOL_REQUEST]>>>
tool_name:「始」NCBIDatasets「末」,
command:「始」search_genomes「末」,
tax_id:「始」511145「末」
<<<[END_TOOL_REQUEST]>>>
返回往往是非常长的学术 JSON 数据流，AI需自行去阅读并浓缩出重点呈现。

【5. PubMedSearch】 (医学文摘学术检索工具)
- tool_name: PubMedSearch
- 功能定位: NCBI 医学论文搜索、详情提取与全文引流核心工具。
- 关键命令(command):
  - search_articles: 常规论文词条搜索，如 `query: cancer treatment`, 可进行最大结果(max_results)限制。
  - get_article_details: 依据 PMID 具体抓出文摘的所有标题、作者与核心摘要结构以便 AI 研究。
  - get_full_text: 依据文献开源的 PMC ID 抽取全文内容给 AI 做深度分析。
- 调用格式:
<<<[TOOL_REQUEST]>>>
tool_name:「始」PubMedSearch「末」,
command:「始」search_articles「末」,
query:「始」advanced neural network biology[Title/Abstract]「末」
<<<[END_TOOL_REQUEST]>>>
- 注意：所有的论文与文摘抓取也极可能含有高负载令牌占用情况，AI务必做好摘要解释而非平铺。

【6. GoogleSearch】 (谷歌搜索 API 版)
- tool_name: GoogleSearch
- 功能定位: 基于 Google Custom Search API 的网页搜索，需要配置 API Key 和搜索引擎 ID。
- 调用格式:
<<<[TOOL_REQUEST]>>>
tool_name:「始」GoogleSearch「末」,
query:「始」搜索关键词「末」
<<<[END_TOOL_REQUEST]>>>

【7. SerpSearch】 (多引擎联合搜索)
- tool_name: SerpSearch
- 功能定位: 基于 SerpApi 的多搜索引擎封装，同时支持 Bing、DuckDuckGo、Google、Google Scholar 四大渠道。
- command 可选: 'bing_search', 'duckduckgo_search', 'google_scholar_search', 'google_search'
- 调用格式:
<<<[TOOL_REQUEST]>>>
tool_name:「始」SerpSearch「末」,
command:「始」google_scholar_search「末」,
q:「始」artificial intelligence in medicine「末」,
as_ylo:「始」2024「末」
<<<[END_TOOL_REQUEST]>>>

【8. VSearch】 (语义并发深度搜索)
- tool_name: VSearch
- 功能定位: 利用小模型内置搜索能力，基于主题和关键词进行多路并行语义检索，返回深度整合结果。
- 调用格式:
<<<[TOOL_REQUEST]>>>
tool_name:「始」VSearch「末」,
SearchTopic:「始」研究主题背景「末」,
Keywords:「始」关键词1, 关键词2, 关键词3「末」,
ShowURL:「始」false「末」
<<<[END_TOOL_REQUEST]>>>

【9. FlashDeepSearch】 (闪电深度研究)
- tool_name: FlashDeepSearch
- 功能定位: 围绕一个主题进行多维度跨领域关键词自动扩展，并发搜索后生成完整的深度研究报告。耗时较长（超时高达 10 分钟）。
- 调用格式:
<<<[TOOL_REQUEST]>>>
tool_name:「始」FlashDeepSearch「末」,
SearchContent:「始」探讨人工智能在气候变化建模中的应用「末」,
SearchBroadness:「始」10「末」
<<<[END_TOOL_REQUEST]>>>

【10. KarakeepSearch】 (Karakeep 书签全文搜索)
- tool_name: KarakeepSearch
- 功能定位: 在 Karakeep 书签服务中进行全文搜索，支持 #tag、is:fav 等高级语法。
- 调用格式:
<<<[TOOL_REQUEST]>>>
tool_name:「始」KarakeepSearch「末」,
query:「始」machine learning is:fav「末」,
limit:「始」5「末」
<<<[END_TOOL_REQUEST]>>>

【11. DeepWikiVCP】 (DeepWiki 代码仓库文档抓取)
- tool_name: DeepWikiVCP
- 功能定位: 从 deepwiki.com 抓取开源代码仓库的全量文档并转为 Markdown 格式返回。
- 调用格式:
<<<[TOOL_REQUEST]>>>
tool_name:「始」DeepWikiVCP「末」,
url:「始」vuejs/vue「末」
<<<[END_TOOL_REQUEST]>>>

【12. Context7】 (代码库实时文档查询)
- tool_name: Context7
- 功能定位: 集成 Context7 平台，为 AI 提供实时的版本特定代码库文档和示例，避免依赖过时训练数据。
- command 可选: 'GetLibraryDocs' (获取特定库文档), 'SearchLibraries' (搜索可用库)
- 调用格式:
<<<[TOOL_REQUEST]>>>
tool_name:「始」Context7「末」,
library:「始」react「末」,
query:「始」hooks「末」
<<<[END_TOOL_REQUEST]>>>

【13. NewsSearch】 (新闻聚合搜索器)
- tool_name: NewsSearch
- 功能定位: 日报工作流专用的多源新闻聚合插件，支持按预定义板块（AI科技、财经、科学、游戏等）自动扫描全网新闻。
- command 可选: 'daily_scan' (全板块自动扫描), 'search' (自定义搜索), 'list_boards' (列出板块)
- 调用格式:
<<<[TOOL_REQUEST]>>>
tool_name:「始」NewsSearch「末」,
command:「始」daily_scan「末」,
boards:「始」ai_tech,finance「末」
<<<[END_TOOL_REQUEST]>>>

【14. PaperReader】 (超文本递归阅读器)
- tool_name: PaperReader
- 功能定位: 将超长 PDF/文档转为目标驱动的多分辨率阅读流程。支持 MinerU 云端高保真解析（保留公式/表格/图片/多栏排版），自动降级到 pdf-parse。
- command 可选:
  - 'IngestPDF': 解析 PDF 并生成分块工件 (需要 filePath 参数)
  - 'Read': 统一自适应阅读（Survey→Triage→DeepDive/Skim→Audit→Synthesize）
  - 'Query': 对已导入文档做检索式问答
- 调用格式:
<<<[TOOL_REQUEST]>>>
tool_name:「始」PaperReader「末」,
command:「始」IngestPDF「末」,
filePath:「始」H:\\books\\paper.pdf「末」
<<<[END_TOOL_REQUEST]>>>
- 注意: 处理大文档（100+ chunks）可能需要数分钟。

【15. KEGGSearch】 (KEGG 生物数据库查询)
- tool_name: KEGGSearch
- 功能定位: 查询 KEGG 数据库的通路、基因、化合物、酶、疾病、药物等多维度生物学数据，提供 30+ 种命令。
- 常用 command:
  - search_pathways / get_pathway_info: 通路搜索与详情
  - search_genes / get_gene_info: 基因搜索与详情
  - search_compounds / get_compound_info: 化合物搜索
  - search_diseases / get_disease_info: 疾病搜索
  - search_drugs / get_drug_info: 药物搜索
  - convert_identifiers: 跨数据库 ID 转换
- 调用格式:
<<<[TOOL_REQUEST]>>>
tool_name:「始」KEGGSearch「末」,
command:「始」search_pathways「末」,
query:「始」glycolysis「末」,
organism_code:「始」hsa「末」
<<<[END_TOOL_REQUEST]>>>

【16. XiaohongshuFetch】 (小红书内容爬取)
- tool_name: XiaohongshuFetch
- 功能定位: 抓取小红书笔记的文本、图片等内容并结构化返回。

【17. CodeSearcher】 (代码库跨文件语义检索)
- tool_name: CodeSearcher
- 功能定位: 在本地代码仓库中进行跨文件的语义或正则检索，辅助代码审查与问题定位。
