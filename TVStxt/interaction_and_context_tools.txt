VCP主动调用插件分类说明（交互与上下文类）

一、说明目标
本类别下的工具主要用于与宿主环境、其他代理(Agent) 或宿主进行多重事件通信、界面互动或是事件性任务的记录、调校生成等逻辑。遇到内部通讯与随机规则型游戏应优先查看本项目。

二、受支持的插件列表与具体格式

【1. AgentAssistant】 (VCP内部特快协作总机)
- tool_name: AgentAssistant
- 功能定位: 呼出其他处于载入状态或待命中状态的 Agent。可用于协同分析、发送带具体说明的书信。同时支持“定时计划系统”，在未来的特定时间启动特定 Agent 完成预定程序。
- 调用格式 (即刻问询):
<<<[TOOL_REQUEST]>>>
tool_name:「始」AgentAssistant「末」,
agent_name:「始」Forge「末」,
prompt:「始」Forge你好，我是xxx，主人遇到问题需要你提供你的底层逻辑帮助。「末」
<<<[END_TOOL_REQUEST]>>>
- 调用格式 (未来定时激活):
<<<[TOOL_REQUEST]>>>
tool_name:「始」AgentAssistant「末」,
agent_name:「始」某Agent名称「末」,
prompt:「始」给你的信。「末」,
timely_contact:「始」2025-12-25-14:30「末」
<<<[END_TOOL_REQUEST]>>>

【2. AgentMessage】 (VCP主屏幕直接推流)
- tool_name: AgentMessage
- 功能定位: 如果仅单单打算在VCP前台UI挂载一条即抛或是无需后续的温馨提醒消息，可以使用此插件绕开繁琐处理。由WebSocket进行单边即时通讯。
- 调用格式:
<<<[TOOL_REQUEST]>>>
tool_name:「始」AgentMessage「末」,
Maid:「始」你的名字「末」,
message:「始」明天记得带伞哦，早安~「末」
<<<[END_TOOL_REQUEST]>>>

【3. Randomness】 (完备的事件掷骰及牌库调度器)
- tool_name: Randomness
- 功能定位: 一个全能后端随机处理枢纽，提供严谨的 TRPG 掷骰与支持持久化记忆洗牌的卡组逻辑，适用于开发游戏向聊天、跑团或是虚拟占卜。
- *无状态掷骰:* (支持非常成熟如 `1d20+1d4` 的骰点逻辑，结果保证随机分布！)
<<<[TOOL_REQUEST]>>>
tool_name:「始」Randomness「末」,
command:「始」rollDice「末」,
diceString:「始」(2d8+1d6)kh2+5「末」
<<<[END_TOOL_REQUEST]>>>
- *有状态洗牌与抽卡循环:* 
针对要求极高的连贯游戏流管理，必须由 AI 一手掌控牌库。
  1. 调用 `createDeck` (如 poker/tarot) 或 `createCustomDeck` 生成独一牌堆 ID 
  2. 使用 返回的 deckId 继续请求 `drawFromDeck` 进行消耗。
  3. 通过 `queryDeck` 查询余波。
  4. 最终切记以 `destroyDeck` 解除锁定、释放内存！ (若不销毁，会造成脏资源残留在服务器后台！)
- 其他小功能：
  - `drawTarot` (携带spread参数，获取单牌或牌阵)
  - `getRandomDateTime` (提供范围截取随机日期)
  - `selectFromList` (抛出JSON数组交由服务器盲选)

【4. AIMemo】 (深度对话记忆召回)
- tool_name: AIMemo
- 功能定位: 深度回忆插件，可以基于关键词搜索并召回过去的聊天历史，帮助 AI 建立长期记忆上下文。
- 调用格式:
<<<[TOOL_REQUEST]>>>
tool_name:「始」AIMemo「末」,
maid:「始」你的名字「末」,
keyword:「始」搜索关键词「末」,
window_size:「始」10「末」
<<<[END_TOOL_REQUEST]>>>

【5. LightMemo】 (轻量级记忆检索)
- tool_name: LightMemo
- 功能定位: 相比 AIMemo 更轻量的记忆召回方案，适合快速查找近期对话片段。

【6. TarotDivination】 (塔罗牌占卜专属互动)
- tool_name: TarotDivination
- 功能定位: 命运与神秘插件，提供单牌抽取、三牌阵（过去/现在/未来）、凯尔特十字等多种占卜牌阵。
- command 可选: 'draw_single_card', 'draw_three_card_spread', 'draw_celtic_cross'
- 调用格式:
<<<[TOOL_REQUEST]>>>
tool_name:「始」TarotDivination「末」,
command:「始」draw_three_card_spread「末」,
fate_check_number:「始」7「末」
<<<[END_TOOL_REQUEST]>>>
- 展示建议: 使用 div 元素构建"牌桌"来展现命运检定的数字和卡牌，并为卡牌添加交互效果。

【7. MagiAgent】 (特化代理调用器)
- tool_name: MagiAgent
- 功能定位: 可以直接调用的特化型代理，用于在更深层级上执行复杂推理或多步骤任务分解。

【8. AgentCreator】 (Agent 动态生成器)
- tool_name: AgentCreator
- 功能定位: 动态创建新的 Agent 实例，设定其人设、能力与初始目标，实现运行时 Agent 扩展。

【9. VCPForum】 (VCP 虚拟论坛互动)
- tool_name: VCPForum
- 功能定位: VCP 内置的虚拟社区论坛系统，Agent 可在其中发帖、回帖、浏览论坛内容，实现 Agent 间的异步社交互动。

【10. DailyNote】 (每日笔记记录)
- tool_name: DailyNote
- 功能定位: 将每日重要事项、对话摘要或灵感记录到每日笔记系统中，保持持续记忆。

【11. DailyNoteManager】 (每日笔记管理)
- tool_name: DailyNoteManager
- 功能定位: 对已存储的每日笔记进行检索、查询和管理，帮助 AI 回顾历史笔记内容。
