## 一、架构概述

GitOperator 是基于**配置档（Profile）驱动**的智能 Git 管理器，通过 `repos.json` 统一管理多仓库的路径、远程地址、凭证和合并策略。

### 核心设计

- **Profile 驱动**：所有操作通过 `profile` 参数指定目标仓库，不指定则使用 `defaultProfile`
- **双远程模型**：每个 Profile 独立配置 `push`（自己的 Fork / origin）和 `pull`（上游原仓库 / upstream）
- **凭证注入**：Push 时自动将 Token 注入 HTTPS URL（仅内存中），操作完成后立即清除，不持久化到 `.git/config`
- **输出脱敏**：所有 git 输出自动过滤 Token/密码，替换为 `***`
- **安全分级**：危险操作需 6 位管理员验证码（requireAdmin）
- **串行调用**：支持 `commandN` 语法在单次调用中执行多条指令

### 配置文件

| 文件 | 说明 |
|------|------|
| GitOperator.js | 主程序（25.37KB） |
| plugin-manifest.json | 插件声明（26 条调用指令） |
| repos.json | 仓库配置档（含凭证，应 gitignore） |
| repos.json.example | 配置档模板（3 种场景示例） |
| config.env | 环境变量 |
| config.env.example | 环境变量模板 |

### repos.json 结构

```json
{
  "defaultProfile": "vcp-server",
  "profiles": {
    "vcp-server": {
      "localPath": "../../",
      "push": { "remote": "origin", "url": "https://github.com/你/VCPServer.git", "branch": "main" },
      "pull": { "remote": "upstream", "url": "https://github.com/RyanLiuCN/VCPServer.git", "branch": "main" },
      "credentials": { "email": "...", "username": "...", "token": "ghp_..." },
      "mergeStrategy": "merge"
    }
  }
}
```

repos.json.example 提供三种场景模板：
- **vcp-server**：Fork 自上游的主仓库（标准双远程 + merge）
- **my-plugin**：独立插件仓库（无上游，pull.url 留空）
- **vcpchat-client**：客户端仓库（绝对路径 + rebase 策略 + dev 分支）

### config.env 配置项

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| PLUGIN_WORK_PATHS | 路径白名单（逗号分隔） | `../../` |
| LOG_LEVEL | 日志级别 | info |
| GIT_EXECUTABLE | Git 路径 | git |
| GIT_TIMEOUT | 命令超时（毫秒） | 30000 |

## 二、完整命令速查表

### 只读查询 🟢

| 命令 | 功能 | 参数 |
|------|------|------|
| **Status** | 仓库状态 | profile(可选) |
| **Log** | 提交历史(结构化JSON) | profile, maxCount(默认20), branch |
| **Diff** | 变更差异(自动截断) | profile, target(如`upstream/main`), maxLines(默认200) |
| **BranchList** | 所有分支 | profile |
| **RemoteInfo** | 远程仓库信息 | profile |
| **StashList** | stash 记录 | profile |
| **TagList** | 标签列表 | profile |
| **ProfileList** | 所有配置档 | 无 |

### 常规操作 🟡

| 命令 | 功能 | 关键参数 |
|------|------|----------|
| **Add** | 暂存文件 | files(必需, 如`.`) |
| **Commit** | 提交变更 | message(必需) |
| **Pull** | 拉取更新 | source(`pull`=上游/`push`=自己,默认`pull`) |
| **Push** | 推送(自动注入凭证) | profile |
| **Fetch** | 获取引用(不合并) | source(`pull`/`push`,默认`pull`) |
| **BranchCreate** | 创建分支 | branchName(必需), startPoint(可选) |
| **Checkout** | 切换分支 | branch(必需) |
| **Merge** | 合并(冲突自动abort) | branch(必需) |
| **Clone** | 克隆仓库(60s超时) | url(必需), localPath(必需) |

### 一键操作 ⭐

| 命令 | 功能 |
|------|------|
| **SyncUpstream** | Fetch→Stash→Merge/Rebase→冲突检测→Stash Pop→Push |

### 危险操作 🔴（需 requireAdmin 验证码）

| 命令 | 功能 | 关键参数 |
|------|------|----------|
| **ForcePush** | 强制推送 | requireAdmin(必需, 验证码) |
| **ResetHard** | 硬重置(含恢复提示) | target(可选), requireAdmin(必需) |
| **BranchDelete** | 删除分支(当前分支保护) | branchName(必需), requireAdmin(必需) |
| **Rebase** | 变基(冲突自动中止) | onto(必需), requireAdmin(必需) |
| **CherryPick** | 摘取提交(冲突自动中止) | commitHash(必需), requireAdmin(必需) |

### 配置档管理

| 命令 | 功能 | 关键参数 |
|------|------|----------|
| **ProfileAdd** | 添加配置档 | profileName(必需), localPath(必需) |
| **ProfileEdit** | 编辑(只传需改字段) | profileName(必需) |
| **ProfileRemove** | 删除配置档 | profileName(必需) |

## 三、VCP 调用格式

⚠️ **命令名必须 PascalCase**（首字母大写）

```
tool_name: GitOperator
command: Status
```

指定配置档：
```
command: Log
profile: vcpchat-client
maxCount: 10
```

Pull 的 source 参数：
```
command: Pull
source: pull      # 从上游拉取（默认）
source: push      # 从自己 Fork 拉取
```

串行调用（commandN 语法）：
```
command1: Status
command2: Log
maxCount2: 5
command3: Diff
target3: upstream/main
```

## 四、标准更新流程

### 阶段 1：预检
```
command: Status          # 查看状态
command: RemoteInfo      # 确认远程配置
command: Log             # 查看本地历史
```

### 阶段 2：同步（推荐一键）
```
command: SyncUpstream
```
自动处理：stash→fetch→merge/rebase→冲突检测→stash pop→push

### 阶段 3：验证
```
command: Status                        # 确认状态
command: Log (maxCount: 15)           # 查看合并历史
command: Diff (target: upstream/main) # 确认与上游一致
```

---

## 五、冲突处理

SyncUpstream / Merge / Rebase / CherryPick 遇到冲突时：
1. 自动执行 `--abort` 安全中止
2. 返回冲突文件列表
3. **不自动解决** → 报告用户手动处理 → `Add` + `Commit` → `Push`

## 六、安全机制

| 机制 | 说明 |
|------|------|
| 路径白名单 | PLUGIN_WORK_PATHS 约束所有操作 |
| Token 脱敏 | sanitizeOutput() 自动过滤凭证 |
| 凭证隔离 | Token 仅内存中注入，不持久化 |
| Auth 守卫 | 5 个危险指令需 requireAdmin 验证 |
| 冲突安全 | 合并/变基冲突时自动 abort |
| 分支保护 | BranchDelete 拒绝删当前分支 |
| 恢复提示 | ResetHard/Rebase 输出 recoveryHint |

---

## 七、Agent 最佳实践

1. **更新前必查 Status**
2. **优先用 SyncUpstream** — 封装完整安全流水线
3. **冲突不硬来** — 报告用户决策
4. **危险操作问 requireAdmin** — 向用户索要验证码
5. **善用 Diff target** — 评估与上游差异
6. **重大操作记日记** — 写入 DailyNote 方便回溯
