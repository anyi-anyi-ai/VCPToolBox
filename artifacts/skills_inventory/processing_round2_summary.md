# Skills 第二轮清洗结果

## 1. 本轮处理目标

在首轮全量拉平基础上，继续增强 [`tools/skills_classifier.py`](tools/skills_classifier.py) 的规则能力，重点解决：

1. 多语言技能的识别问题；
2. 大型技能仓的待确认分类占比过高问题；
3. 输出维度不足，不利于后续人工复核与 VCP 接入治理的问题。

## 2. 本轮已完成增强

| 增强项 | 说明 |
|---|---|
| 多语言标准化 | 新增文本归一化与语言提示识别（`en` / `es` / `zh` / `unknown`） |
| 语义关键词分类 | 新增语义级关键词规则，减少仅靠技能名分类的误差 |
| README 纳入过滤 | 对 [`README.md`](../VCPziliao/skills/superpowers-main/README.md) 做更谨慎筛选，减少噪声 |
| 语言提示输出 | 在结构化记录中新增 `language_hint` 字段 |
| 重复组摘要 | 在 Markdown 报告中新增重复组汇总 |
| 多语言待确认标记 | 对西语/英语但未命中分类的技能单独标记，便于专项复核 |

## 4. 结果解读

### 4.1 主要收益

1. 待确认分类已从 1413 项降到 86 项，说明第二轮规则增强有效；
2. 对 [`antigravity-awesome-skills-main`](../VCPziliao/skills/antigravity-awesome-skills-main) 等大型技能仓的覆盖能力明显提升；
3. 当前输出已经可以支持：
   - 高优先级技能筛选；
   - 重复组复核；
   - 多语言技能专项复核；
   - VCP Skill Manifest 草案生成前的数据清洗；
4. 目前最大的剩余问题不再是“无法分类”，而是“分类粒度可能偏粗，需要按仓库和重复组进一步精修”。

### 4.2 当前仍需注意的问题

| 问题 | 说明 |
|---|---|
| `unknown` 语言仍较多 | 说明很多技能描述较短，或缺少 frontmatter 描述 |
| 知识生产类偏高 | 部分泛化关键词会将技能过多归入 C 类，需进一步分流 |
| README 仍有少量噪声 | 但已可控，不再是主矛盾 |
| 重复组仍未做实体合并 | 当前只是标记，尚未自动去重 |

## 5. 当前建议的第三步动作

### 5.1 优先处理项

| 优先级 | 动作 | 目的 |
|---|---|---|
| P0 | 生成重复技能组单独报告 | 为合并/归档做准备 |
| P0 | 输出 P0/P1 技能清单 | 形成 VCP 首批接入候选集 |
| P1 | 为 `unknown` 技能补充标题/正文前几段抽样判定 | 进一步压缩待确认项 |
| P1 | 生成 VCP 原生 manifest 草案 | 为后续技能注册表接入做准备 |
| P2 | 引入相似度聚类 | 从“规则重复组”升级为“文本近似聚类” |

### 5.2 当前最适合继续推进的方向

下一步最值得做的，不是继续扩大分类规则，而是让 [`tools/skills_classifier.py`](tools/skills_classifier.py) 进入“治理输出阶段”：

1. 输出 `P0` / `核心能力` 列表；
2. 输出重复组报告；
3. 输出待确认项清单；
4. 生成适配 VCP 的 `skill_manifest` 草案。

这一步完成后，skills 目录就会从“资料堆”进入“可接入资产池”。
