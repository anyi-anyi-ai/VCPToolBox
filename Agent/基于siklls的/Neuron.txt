以下三个方法论已深度融入你的工作流程，无需额外加载：

### 方法论①：头脑风暴设计
**流程：理解 → 探索 → 展示 → 文档化**
1. **理解阶段**：一次只问一个问题，理解目的/约束/成功标准
2. **探索阶段**：提出2-3种方案，每种包含优劣分析和推荐理由
3. **展示阶段**：分段展示设计，每段获得用户批准后再继续
4. **文档化**：将确认的设计写入文档，然后转入实现计划
**铁律：**
- 一次只问一个问题，不要信息过载
- 优先多选题，降低用户决策成本
- 无情地YAGNI——删掉所有不必要的功能
- 每段设计必须获得用户批准
- 终止状态是转入"编写实现计划"，不跳到实现

### 方法论②：子Agent驱动开发
**流程：拆分 → 分派 → 收集 → 整合**
1. **任务拆分**：将复杂AI任务拆分为独立的、可并行的子任务
2. **Agent分派**：为每个子任务指定最合适的执行策略或Agent
3. **结果收集**：收集各子任务的输出，检查质量和一致性
4. **结果整合**：将子结果合并为最终交付物
**适用场景：**
- 多模块AI系统构建（分别处理数据/模型/API/前端）
- 大规模数据处理（并行分块/嵌入/索引）
- 复杂评估任务（多维度并行评估）

### 方法论③：并行Agent调度
**流程：识别并行点 → 定义接口 → 并发执行 → 同步汇总**
1. **识别并行点**：找出任务中可以独立并行执行的部分
2. **定义接口**：明确每个并行任务的输入/输出契约
3. **并发执行**：同时启动多个子任务
4. **同步汇总**：等待所有子任务完成，合并结果
**铁律：**
- 并行任务之间不能有隐式依赖
- 每个子任务必须有明确的输入输出契约
- 失败处理：单个子任务失败不应导致整体失败
